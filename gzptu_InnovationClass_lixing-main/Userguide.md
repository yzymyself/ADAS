## 1. 项目概述
ADASSim_v2 是一款面向教学场景的 **ADAS（高级驾驶辅助系统）仿真实训平台**，基于 **CARLA 0.9.13** 作为仿真后端，通过 **PyQt5** 构建桌面交互界面。系统支持以下核心能力：

| 能力 | 说明 |
| --- | --- |
| **ADAS 算法仿真** | ACC、AEB、LKA、BSD 四大 ADAS 功能的参数化仿真验证 |
| **G29 方向盘联动** | Logitech G29 方向盘+踏板+H档排挡杆的全物理操作 |
| **自动驾驶演示** | 基于 CARLA 路径规划的全自动驾驶，力反馈方向盘同步 |
| **SLAM 建图** | 激光雷达实时 SLAM + 高精地图采集 |
| **硬件传感器** | 毫米波雷达、超声波雷达的真实数据采集与可视化 |
| **测试车辆管理** | 动态添加/删除同车道、邻车道的测试车辆 |
| **故障注入教学** | 传感器、执行器故障模拟 |
| **双屏显示** | 主屏仿真画面 + 副屏传感器监控 |


### 系统架构简图

┌──────────────┐    ┌──────────────────────────────────────┐

                    │ G29 方向盘                    │ 毫米波/超声波雷达

## 2. 快速开始
### 2.1 启动前检查
| 检查项 | 要求 |
| --- | --- |
| CARLA 服务端 | `CarlaUE4.exe` 已启动，场景加载完成 |
| G29 方向盘 | 已连接 USB，Logitech G HUB 正在运行 |
| 串口设备 | CH341 适配器已插入（如需硬件传感器） |
| Python 环境 | conda 环境 `torch241_cu124` 已激活 |


### 2.2 启动步骤
**方式一：双击启动**

```plain
双击项目根目录下的 run.bat
```

**方式二：命令行启动**

```bash
cd E:\ADAS\ADASSim_v2
E:\Anaconda3\envs\torch241_cu124\pythonw.exe -m app.main
```

### 2.3 首次运行流程
1. 启动后出现 **主窗口** + **监控浮窗**（双屏时自动分配到副屏）
2. 首页展示 4 个功能卡片：ADAS 仿真 / 自动驾驶 / 建图导航 / 硬件系统
3. 选择功能 → 配置参数 → 点击"启动仿真"开始

## 6. ADAS 功能详解
### 6.1 ACC 自适应巡航
**功能**：自动调节车速与前车保持安全距离。

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| 目标速度 | 60 km/h | 无前车时的巡航速度 |
| 跟车距离 | 80 m | 与前车的目标距离 |
| 时距 | 2.5 s | 动态跟车距离的时间参数 |
| 速度 P 增益 | 0.3 | 速度控制比例系数 |
| 速度 D 增益 | 0.15 | 速度控制微分系数 |
| 最大油门 | 0.8 | 自动油门上限 |
| 最大刹车 | 0.6 | 自动刹车上限 |


**激活与退出**：

+ **激活**：按左拨片
+ **退出**：
    - 主动踩刹车（退出）
    - 主动踩油门（退出）
    - 按左拨片关闭

**驾驶模式**：激活时自动切换为自动挡，退出后恢复原档位设置。

### 6.2 AEB 自动紧急制动
**功能**：基于 TTC（碰撞时间）的前方碰撞预警与自动制动。

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| 预警 TTC | 2.5 s | 触发碰撞预警的时间阈值 |
| 制动 TTC | 2.0 s | 触发全力制动的时间阈值 |
| 制动力 | 0.9 | 全力制动时的刹车值 |
| 最小安全距离 | 5.0 m | 与前车的最小安全距离 |
| FOV 角度 | 12° | 关注的前方角度范围 |


**AEB 状态机**：

```plain
NORMAL ──TTC<预警──► WARNING ──TTC<制动──► BRAKE ──危险解除──► HOLD
   ▲                                                              │
   └──────────────── 冷却期(2.5s) 或 前车距离>8m ◄────────────────┘
```

**冷却期（COOLDOWN）**：制动释放后进入 2.5s 冷却期，此期间 AEB 灵敏度降低，避免频繁触发。

**前车检测**：优先使用 CARLA Waypoint API 的 Ground Truth 车道级检测，辅以雷达聚类。

### 6.3 LKA 车道保持辅助
**功能**：自动修正方向盘使车辆保持在车道中心。

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| 偏移阈值 | 0.25 m | 偏离车道中心触发修正的距离 |
| 转向增益 | 0.8 | 偏移量→转向值的转换系数 |
| 航向增益 | 0.4 | 航向角差→转向值的增益 |
| 最大转向 | 0.3 | 自动修正的最大转向角 |
| 预瞄距离 | 15 m | 向前预瞄的距离 |


**激活与退出**：

+ **激活**：按右拨片
+ **退出**：按右拨片

**G29 力反馈**：LKA 激活时，方向盘自动施加修正力矩。

### 6.4 BSD 盲区检测
**功能**：检测后侧方盲区内的接近车辆并发出警告。

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| 检测范围 | 5.0 m | 雷达盲区检测距离 |
| 盲区角度 | 60° | 盲区检测角度 |
| 最低接近速度 | 3 km/h | 低于此速度不报警 |
| 最高接近速度 | 150 km/h | 超过此速度视为静止墙体 |
| 警报帧数 | 2 | 连续检测帧数阈值 |


**警告逻辑**：仅当驾驶员有同侧转向意图时（方向盘偏转 > 阈值），才触发BSD警告。

### 6.5 ACC+LKA 联合模式
同时启用 ACC 和 LKA，实现横向+纵向的辅助驾驶。

+ ACC 控制油门/刹车（纵向）
+ LKA 控制方向盘（横向）
+ 退出规则遵循各自逻辑

## 10. 常见问题
### Q1：启动后提示 CARLA 连接失败？
**原因**：`CarlaUE4.exe` 未启动或未加载完成。  
**解决**：先启动 CARLA，等待 30~60 秒场景加载完毕后再点击启动。

### Q2：G29 方向盘无反应？
**原因**：Logitech G HUB 未运行或 G29 未被识别。  
**解决**：

1. 确保 G HUB 正在运行
2. 检查 USB 连接
3. 在硬件页面查看 G29 状态
4. 尝试重新插拔 USB

### Q3：LKA 力反馈不工作？
**原因**：G HUB 未运行或 DLL 初始化失败。  
**解决**：

1. 确保 G HUB 正在运行
2. 确认 `LogitechSteeringWheelEnginesWrapper.dll` 在项目根目录
3. 在车道线页面点击"重连 G29"

### Q4：手动挡总是熄火？
**原因**：离合操作不当或速度档位不匹配。  
**解决**：

1. 起步时缓慢抬离合
2. 注意各档位速度范围
3. 超速时及时升档
4. 可在设置中切换为自动挡模式

### Q5：如何保存自定义的 ADAS 参数？
直接编辑 `config/adas_params.yaml` 文件，下次启动即生效。UI 中的"恢复默认"按钮会还原到此文件的值。

### Q6：双屏显示异常？
**原因**：显示器设置问题。  
**解决**：

1. Windows 显示设置中确认双屏为"扩展"模式
2. 重新启动程序

### Q7：运行 `run.bat` 报 `ModuleNotFoundError`？
**原因**：conda 环境未激活或路径不匹配。  
**解决**：

1. 确认 conda 环境 `torch241_cu124` 已创建
2. 确认 `run.bat` 中 `pythonw.exe` 路径正确
3. 确认所有依赖已安装（参考 `docs/requirements.txt`）

### Q8：车道线识别无画面？
**原因**：USB 摄像头未连接或被其他程序占用。  
**解决**：

1. 检查摄像头 USB 连接
2. 关闭其他使用摄像头的程序
3. 在设备管理器中确认摄像头正常
